Audio stack stage 7

Este stack sigue separado del video para que el flujo de audio crezca sin tocar MediaMTX ni sus rutas.

En esta etapa cada streaming de audio mantiene su propio input live, sus mounts dedicados en Icecast y su salida HLS interna, pero ahora comparte un AutoDJ por compañia para el fallback musical.

  • Healthcheck: /healthz
  • Status: /status
  • HLS per stream: /hls/<streamingAlias>/<publishKey>/live.m3u8
  • MP3 per stream: /listen/<streamingAlias>/<publishKey>/radio.mp3
  • AAC per stream: /listen/<streamingAlias>/<publishKey>/radio.aac
  • Live publish per stream: /live/<publishMountToken>
  • AutoDJ active file per company: /srv/audio/playlists/companies/<companyId>/active.m3u

Prioridad de reproduccion: live gana siempre; si no hay live, primero se intenta la playlist personalizada activa de la compañia; si no hay horario custom activo, se usa la playlist default 24/7.

La libreria, carpetas, playlists y horarios se administran desde la pagina AutoDJ del frontend. Si dos horarios custom se superponen, el backend rechaza el cambio y explica el conflicto.
